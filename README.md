# Hackathon Data & Models

Sample financial data and pre-staged models for the HP + Morningstar AI Hackathon. The data was generated by `prep_data.py` and is completely self-contained - no internet access is needed during the hackathon.

---

## What's in here

### `portfolios/`
Five sample equity portfolios as JSON files. Each contains 15-35 holdings with ticker, company name, sector, industry, weight, shares, market value, and price.

| Portfolio | Style | Notes |
|-----------|-------|-------|
| `alpha_growth_fund.json` | Growth | Heavy tech concentration (~45%) - intentionally exceeds the 30% sector limit |
| `beta_value_fund.json` | Value | Well-diversified across sectors |
| `gamma_income_fund.json` | Income | Healthcare-biased (~35%) - also over the 30% limit |
| `delta_balanced_fund.json` | Balanced | Broad allocation, 35 holdings |
| `epsilon_concentrated_fund.json` | Concentrated | Financial Services-heavy (~40%), only 15 holdings |

Alpha, Gamma, and Epsilon are designed to trigger compliance violations. Beta and Delta are generally clean.

### `filings/`
SEC 10-K filing full text from 10 public companies (AAPL, MSFT, JPM, JNJ, XOM, PG, UNH, MA, NVDA, KO), pulled directly from EDGAR via `edgartools`. Each file is a raw `.txt` file containing the complete extracted filing text.

These are real filings. The content covers business descriptions, risk factors, financial statements, capital expenditures, and regulatory disclosures - exactly the kind of content a financial research agent would need to search.

**Chunking is deliberately left to participants.** How you split these documents for embedding - by character count, sentence boundary, paragraph, filing section, or something else - will directly affect retrieval quality. Chunk size, overlap strategy, and whether to preserve structural boundaries (like Item headers) are all decisions worth experimenting with.

### `compliance/`
- **`restricted_list.csv`** - 15 securities flagged for various compliance reasons (MNPI, insider trading investigation, earnings blackout, OFAC review, etc.). Includes real tickers from the portfolio universe plus synthetic ones. Each entry has a restriction type (`hard_block`, `soft_flag`, or `watch_list`), a reason, effective/expiry dates, and who added it.
- **`compliance_rules.yaml`** - Policy rules the agent can reference programmatically: sector concentration limits (30%), single-holding limits (5%), top-5 cap (35%), Herfindahl threshold (0.15), minimum holdings (15), and audit trail requirements.

### `benchmark/`
- **`benchmark_questions.json`** - 10 natural language compliance queries at easy/medium/hard difficulty. Each includes the expected tool calls, what the answer should contain, and whether a compliance violation should be flagged. Use these in Sprint 3 to evaluate agent accuracy.

### `metadata/`
- **`sector_map.json`** - Ticker-level metadata (company name, sector, industry, market cap, price) pulled from Yahoo Finance.
- **`filing_manifest.json`** - Index of which filings were pulled, from which companies, how many chunks each has, and the filing date.
- **`data_manifest.json`** - Full manifest of the data directory with generation timestamp, sources, and usage notes.

## Data sources

| Data | Source | Cost |
|------|--------|------|
| Ticker metadata (sectors, prices) | Yahoo Finance via `yfinance` | Free, no API key |
| SEC 10-K filings | SEC EDGAR via `edgartools` | Free, no API key |
| Portfolios | Synthetically generated using real ticker/sector data | - |
| Restricted list | Synthetically generated | - |
| Compliance rules | Synthetically generated (realistic but fictional) | - |

---

## Important notes

- The portfolios are **deliberately non-compliant** in specific ways so the agent has real violations to find. This is by design.
- The restricted list includes some tickers that also appear in the portfolios. This creates cross-reference scenarios for the `check_restricted_list` tool.
- All dates, weights, and values were generated at prep time and are static. They won't update during the hackathon - that's the point.
